옵시디언 플러그인 방식은 LLM 도구와 함께 사용할 수도 있습니다.  
옵시디언 보관함 폴더를 LLM 도구가 함께 볼 수 있게 하면, 위키독스 원고 파일을 AI 도움을 받아 다듬을 수 있습니다.

## LLM을 함께 사용하는 이유

LLM은 원고 작성의 보조 도구로 사용할 수 있습니다.  
마크다운 원고를 읽고 문장 개선, 구조 제안, 표현 통일, 중복 설명 점검 같은 작업을 도와줍니다.

옵시디언은 원고를 읽고 정리하는 데 편하고, LLM 도구는 여러 파일을 함께 살펴보며 수정 제안을 받는 데 편합니다.  
두 흐름을 함께 쓰면 옵시디언에서 책의 흐름을 확인하고, LLM 도구로 문장과 구조를 다듬는 식으로 역할을 나눌 수 있습니다.

사용할 수 있는 도구는 하나로 정해져 있지 않습니다.  
Cursor처럼 편집기 안에 AI 기능이 들어 있는 도구를 사용할 수도 있고, Codex나 Claude Code처럼 터미널에서 실행하는 도구를 사용할 수도 있습니다.

## 작업 환경 준비하기

먼저 옵시디언에서 위키독스 플러그인을 설치하고, 작성할 책을 내려받아 둡니다.  
그다음 사용할 LLM 도구에서 같은 옵시디언 보관함 폴더를 엽니다.

이때 중요한 것은 옵시디언과 LLM 도구가 같은 폴더를 보고 있어야 한다는 점입니다.  
옵시디언에서 내려받은 책 파일과 LLM 도구에서 여는 파일이 서로 다르면 수정 내용이 엇갈릴 수 있습니다.

기본 흐름은 다음과 같습니다.

1. 옵시디언에서 위키독스 책을 내려받습니다.
2. LLM 도구에서 옵시디언 보관함 폴더를 엽니다.
3. 원고를 수정하거나 AI에게 편집 도움을 요청합니다.
4. 옵시디언에서 수정 결과를 다시 읽고 확인합니다.
5. 문제가 없다면 옵시디언의 `위키독스 보내기`로 변경 내용을 반영합니다.

## AI로 원고 다듬기

LLM 도구에서는 한 파일만 보면서 문장을 고칠 수도 있고, 여러 파일을 함께 확인하며 흐름을 점검할 수도 있습니다.  
예를 들어 다음과 같은 작업을 요청할 수 있습니다.

```text
이 장의 문장이 너무 딱딱한 부분을 찾아 자연스럽게 다듬어 주세요.
```

```text
04장 전체에서 같은 설명이 반복되는 부분을 찾아 주세요.
```

```text
이 문서의 소제목 순서가 독자 입장에서 자연스러운지 검토해 주세요.
```

AI가 제안한 내용은 바로 반영하기보다 직접 읽어 보고 선택적으로 적용하는 것이 좋습니다.  
책의 문체와 설명 수준은 저자가 최종적으로 판단해야 합니다.

## 동기화 전에 확인할 것

LLM 도구로 파일을 수정한 뒤에는 옵시디언에서 해당 파일을 다시 열어 봅니다.  
마크다운 문법, 이미지 링크, 제목 구조가 깨지지 않았는지 확인합니다.

그다음 `위키독스 보내기`를 실행해 위키독스에 반영합니다.  
반영 후에는 위키독스 책 화면에서 실제 독자에게 보이는 결과를 확인합니다.

## 주의할 점

AI 도구와 동기화 플러그인을 함께 사용할 때는 변경 범위가 커지기 쉽습니다.  
한 번에 많은 파일을 수정하기보다 한 장씩 나누어 작업하는 것이 안전합니다.

API 토큰이나 개인 계정 정보가 원고에 포함되지 않도록 주의합니다.  
AI에게 파일 전체를 보여 주거나 여러 파일을 수정하게 할 때는 민감한 정보가 들어 있지 않은지 먼저 확인합니다.

또한 옵시디언과 LLM 도구를 동시에 열어 같은 파일을 수정할 때는 저장 순서에 주의해야 합니다.  
한쪽에서 수정한 내용을 다른 쪽이 오래된 상태로 덮어쓰지 않도록, 수정 후에는 파일을 다시 열거나 새로고침하여 최신 상태를 확인합니다.
