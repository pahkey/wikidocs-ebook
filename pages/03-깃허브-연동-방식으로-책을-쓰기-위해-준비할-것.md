앞 장에서는 위키독스에서 책을 작성하는 여러 방법을 살펴보았습니다.  
이 책에서는 그중에서도 깃허브와 연동하여 원고를 관리하고, 옵시디언과 LLM 도구를 함께 활용하는 방식으로 진행합니다.

이 방식으로 책을 쓰려면 몇 가지 준비물이 필요합니다.  
처음부터 모든 도구를 완벽하게 익힐 필요는 없습니다. 먼저 어떤 도구가 어떤 역할을 하는지 이해하고, 하나씩 준비해 나가면 됩니다.

[TOC]

## 위키독스 계정

가장 먼저 필요한 것은 위키독스 계정입니다.  
위키독스 계정은 책을 공개하고, 독자에게 보여 줄 출판 공간을 준비하기 위해 필요합니다.

먼저 위키독스에 가입해 둡니다.  
실제 책 생성과 깃허브 저장소 연결은 04장에서 진행합니다.

## 깃허브 계정과 Git

두 번째로 필요한 것은 깃허브 계정입니다.  
깃허브는 원고 파일을 보관하고, 수정 이력을 관리하는 공간으로 사용합니다.

로컬 컴퓨터에서 저장소를 내려받고 변경 내용을 올리려면 Git도 필요합니다.  
지금은 깃허브 계정과 Git 사용 환경을 준비하는 것이 목적입니다.

## 옵시디언

세 번째 준비물은 옵시디언입니다.  
옵시디언은 마크다운 문서를 편집하고 관리하기 좋은 노트 앱입니다.

설치와 기본 개념만 먼저 익히고, 실제 원고 저장소를 여는 과정은 뒤에서 진행합니다.

## LLM 도구

마지막으로 준비할 것은 Codex나 Claude Code 같은 LLM 도구입니다.  
이 도구들은 원고를 대신 완성해 주는 도구라기보다는, 책을 쓰는 과정을 도와주는 협업 도구에 가깝습니다.

여기서는 사용할 도구를 정하고 실행 환경을 준비하는 데 집중합니다.

## 준비가 끝나면

이 준비가 끝나면 깃허브와 연동하여 위키독스 책을 작성할 기본 환경이 갖추어집니다.  
각 준비물을 하나씩 확인한 뒤, 04장에서 이 환경을 바탕으로 실제 책을 만들어 보겠습니다.
