이 책에서는 그중에서도 깃허브와 연동하여 원고를 관리하고, 옵시디언과 LLM 도구를 함께 활용하는 방식으로 진행합니다.

이 방식으로 책을 쓰려면 몇 가지 준비물이 필요합니다.  
처음부터 모든 도구를 완벽하게 익힐 필요는 없습니다. 먼저 어떤 도구가 어떤 역할을 하는지 이해하고, 하나씩 준비해 나가면 됩니다.

## 위키독스 계정

가장 먼저 필요한 것은 위키독스 계정입니다.  
위키독스 계정은 책을 만들고, 원고를 공개하고, 독자와 만나는 공간을 준비하기 위해 필요합니다.

이 책에서는 원고 작성, 목차 구성, 이미지 관리 같은 작업을 주로 깃허브 저장소에서 진행합니다.  
위키독스는 깃허브 저장소와 연결되어, 저장소에 있는 원고를 독자가 읽을 수 있는 책으로 보여 주는 역할을 합니다.

따라서 위키독스에서는 원고를 직접 자주 고치기보다는 책의 공개 여부, 페이지 공개 여부, 광고 표시 여부 같은 운영 설정을 주로 관리하게 됩니다.  
원고는 깃허브에서 작성하고, 위키독스는 그 결과를 독자에게 전달하는 출판 공간으로 사용하는 흐름입니다.

## 깃허브 계정

두 번째로 필요한 것은 깃허브 계정입니다.  
깃허브는 원고 파일을 보관하고, 수정 이력을 관리하는 공간으로 사용합니다.

깃허브를 사용하면 책의 각 페이지를 마크다운 파일로 관리할 수 있습니다.  
문장을 고치거나 새 장을 추가할 때마다 변경 내용이 기록되기 때문에, 책이 어떻게 발전해 왔는지 확인하기 좋습니다.

위키독스와 깃허브를 연동하면 깃허브 저장소에 있는 원고를 위키독스 책으로 반영할 수 있습니다.  
이 책에서는 이 흐름을 기준으로 원고를 작성하고 공개하는 과정을 진행합니다.

## 옵시디언

세 번째 준비물은 옵시디언입니다.  
옵시디언은 마크다운 문서를 편집하고 관리하기 좋은 노트 앱입니다.

깃허브 저장소를 옵시디언 보관함처럼 열어 두면 책의 원고를 로컬에서 편하게 편집할 수 있습니다.  
각 장을 파일로 살펴보고, 목차 구조를 확인하고, 여러 페이지를 오가며 내용을 다듬기에 좋습니다.

옵시디언은 단순한 글쓰기 도구를 넘어 원고 작업 공간이 됩니다.  
자료를 정리하고, 초안을 만들고, 완성된 문장을 책의 구조에 맞게 배치하는 흐름을 자연스럽게 이어 갈 수 있습니다.

## LLM 도구

마지막으로 준비할 것은 Codex나 Claude Code 같은 LLM 도구입니다.  
이 도구들은 원고를 대신 완성해 주는 도구라기보다는, 책을 쓰는 과정을 도와주는 협업 도구에 가깝습니다.

LLM 도구를 사용하면 아이디어를 초안으로 바꾸거나, 이미 작성한 문장을 더 읽기 좋게 다듬거나, 여러 페이지의 표현을 일관되게 맞출 수 있습니다.  
혼자서 막히기 쉬운 부분을 넘어서고, 원고를 더 빠르게 개선하는 데 도움이 됩니다.

이 책에서는 옵시디언에 터미널 플러그인을 설치하고, 그 안에서 Codex나 Claude Code 같은 도구를 사용하는 흐름을 다룰 예정입니다.  
원고를 보고 있는 같은 작업 공간에서 LLM 도구를 실행하면, 책의 구조를 살피며 필요한 수정을 바로 진행할 수 있습니다.

## 준비가 끝나면

위키독스 계정은 책이 독자에게 보이는 공간을 준비합니다.  
깃허브 계정은 원고와 수정 이력을 관리합니다.

옵시디언은 실제로 글을 쓰고 정리하는 작업 공간이 됩니다.  
LLM 도구는 초안 작성과 문장 수정, 구조 정리를 도와주는 조력자가 됩니다.

이 네 가지를 준비하면 깃허브와 연동하여 위키독스 책을 작성할 기본 환경이 갖추어집니다.  
다음 장부터는 이 준비물을 하나씩 실제로 설정하면서, 위키독스 책을 만들기 위한 작업 환경을 만들어 보겠습니다.
