새 책을 만든 뒤에는 책 전체 설정을 수정하는 화면과 개별 페이지를 수정하는 화면을 구분해서 사용합니다.  
책의 제목이나 공개 여부는 책 수정에서 관리하고, 실제 원고 내용은 페이지 수정에서 관리합니다.

## 책 수정과 페이지 수정의 차이

책 수정은 책 전체에 적용되는 설정을 관리하는 화면입니다.  
책 제목, 책 이미지, 공개 여부, 책 설명, 저작권, 광고 설정처럼 책 전체에 영향을 주는 항목을 수정합니다.

페이지 수정은 각 장의 제목과 내용을 관리하는 화면입니다.  
페이지 제목, 부모 페이지, 공개 여부, 이미지, 본문 내용을 수정할 때 사용합니다.

## 수정 화면으로 들어가기

책이나 페이지를 수정하려면 먼저 독자가 보는 읽기 화면에서 시작합니다.  
위키독스에 로그인한 상태로 내가 관리할 수 있는 책이나 페이지를 열면, 화면 오른쪽 위에 수정 화면으로 이동하는 버튼이 표시됩니다.

책 전체 설정을 바꾸려면 책 읽기 화면에서 오른쪽 위의 수정 버튼을 선택합니다.  
이 버튼을 누르면 책 수정 화면으로 이동합니다.

![](../assets/Pasted%20image%2020260429094828.png)

페이지 내용을 바꾸려면 수정할 페이지의 읽기 화면을 먼저 엽니다.  
그다음 화면 오른쪽 위의 수정 버튼을 선택하면 페이지 수정 화면으로 이동합니다.

![](../assets/Pasted%20image%2020260429094846.png)

수정 화면에 들어간 뒤에는 왼쪽 목록에서 책 제목이나 개별 페이지를 선택해 수정 대상을 바꿀 수도 있습니다.  
책 제목을 선택하면 책 수정 화면으로 이동하고, 개별 페이지를 선택하면 페이지 수정 화면으로 이동합니다.

![](../assets/Pasted%20image%2020260429093850.png)

## 책 수정에서 관리하는 항목

책 수정 화면에서는 책 전체에 적용되는 설정을 탭별로 관리합니다.  
각 탭은 책의 기본 정보, 운영 방식, 백업, 전자책 신청처럼 페이지 하나가 아니라 책 전체에 영향을 주는 항목을 다룹니다.

![](../assets/Pasted%20image%2020260429093936.png)

### 기본

기본 탭에서는 책의 제목, 이미지, 공개 여부, 책 설명(요약)을 설정합니다.  
책 제목은 독자가 책 목록과 책 보기 화면에서 확인하는 이름이고, 책 이미지는 책을 대표하는 표지 이미지로 사용됩니다.

책 설명은 책 보기 화면에 표시되는 소개 문장입니다.  
어떤 내용을 다루는 책인지, 어떤 독자에게 도움이 되는지 짧게 정리합니다.

![](../assets/Pasted%20image%2020260429152616.png)
### 저작권

저작권 탭에서는 책의 저작권 표시 방식을 정합니다.  
CCL 방식으로 선택할 수도 있고, 라이선스 이름과 URL을 직접 입력할 수도 있습니다.

공개 책이라면 독자가 원고를 어디까지 이용할 수 있는지 알 수 있도록 저작권 정보를 명확히 적어 두는 것이 좋습니다.  
특히 공동 집필이나 전자책 판매를 생각하고 있다면 초기에 저작권 설정을 확인해 둡니다.

![](../assets/Pasted%20image%2020260429152629.png)
### 공동 저자

공동 저자 탭에서는 함께 책을 관리할 저자를 추가하거나 제거합니다.  
이름이나 이메일로 사용자를 검색해 공동 저자로 등록할 수 있습니다.

공동 저자는 책의 원고와 설정을 함께 다룰 수 있으므로 신뢰할 수 있는 사람만 추가합니다.  
책을 만든 메인 저자는 변경할 수 없으므로 처음 책을 만들 때 사용할 계정도 신중하게 정해야 합니다.

![](../assets/Pasted%20image%2020260429152643.png)

### 광고

광고 탭에서는 책에 광고를 표시할지 정합니다.  
광고 표시 여부, 중간 광고 사용 여부를 설정할 수 있습니다.

책을 공개한 뒤 광고 수익을 고려한다면 이 탭에서 광고 설정을 확인합니다.  
아직 원고를 작성하는 중이거나 비공개로 운영하는 책이라면 나중에 설정해도 됩니다.

![](../assets/Pasted%20image%2020260429152707.png)

### 사용자 스타일/스크립트

사용자 스타일/스크립트 탭에서는 책 보기 화면에 적용할 CSS나 스크립트를 설정합니다.  
기본 화면만으로 부족할 때 글꼴, 간격, 색상, 특별한 표시 방식을 조정하는 용도로 사용할 수 있습니다.

이 기능은 책 전체의 표시 방식에 영향을 주므로 필요한 경우에만 사용합니다.  
전자책이나 모바일 화면에서는 의도와 다르게 보일 수 있으므로 적용 후 실제 책 화면에서 확인해야 합니다.

![](../assets/Pasted%20image%2020260429152814.png)

### 기타

기타 탭에서는 줄 바꿈, 정렬 방식, HTML 사용 여부, 댓글 알림, 목차 폴딩 같은 부가 설정을 관리합니다.  
엔터키 줄 바꿈을 켜면 줄 끝에 공백 두 칸을 넣지 않아도 줄 바꿈이 적용됩니다.

역순 정렬은 목차를 제목의 역순으로 정렬하는 설정입니다.  
일반적인 책에서는 보통 번호를 붙여 순서를 관리하고, 필요한 경우에만 역순 정렬을 사용합니다.

목차 폴딩을 사용하면 하위 페이지가 있는 목차를 접고 펼칠 수 있습니다.  
장이 많고 하위 페이지가 많은 책이라면 독자가 목차를 훑어보기 편해집니다.

![](../assets/Pasted%20image%2020260429152857.png)

### 책 백업

책 백업 탭에서는 현재 책의 내용을 백업합니다.  
백업 파일을 내려받거나, 깃허브와 연결해 원고와 이미지를 보관할 수 있습니다.

큰 수정을 하기 전이나 책 삭제 같은 위험한 작업을 하기 전에는 백업을 먼저 만들어 두는 것이 좋습니다.  
백업은 원고를 되돌려야 할 때 사용할 수 있는 안전장치입니다.

![](../assets/Pasted%20image%2020260429152948.png)

### 책 삭제

책 삭제 탭에서는 책을 완전히 삭제합니다.  
삭제할 때는 책 이름을 다시 입력해 삭제 의사를 확인합니다.

책 삭제는 되돌릴수 없습니다.  
실행하기 전에 백업을 만들었는지, 정말 삭제해도 되는 책인지 반드시 확인합니다.

![](../assets/Pasted%20image%2020260429153021.png)

### 전자책

책 작성 후 전자책으로 판매하기를 희망할 경우 이 탭에서 전자책 판매를 신청할 수 있습니다.  
신청할 때는 이메일, 판매 방식, 판매 희망 금액, AI 사용 비율, 하고 싶은 말 등을 입력합니다.

전자책 판매 신청은 `전자책 검증`을 통과해야 진행할 수 있습니다.  
검증을 통과하지 못하면 신청할 수 없으므로, 안내되는 오류를 확인하고 책 내용을 수정한 뒤 다시 검증합니다.

신청이 승인되면 `전자책 개정` 버튼이 표시됩니다.  
전자책 개정은 현재 변경된 책 내용을 판매 중인 전자책에 반영하는 작업입니다.

전자책 개정은 특정 시점의 책 내용을 저장하는 스냅샷과 비슷합니다.  
책 본문을 수정했더라도 전자책 개정을 하지 않으면, 마지막으로 개정된 내용을 기준으로 전자책이 판매됩니다.

![](../assets/Pasted%20image%2020260429153114.png)

## 페이지와 목차 수정하기

페이지 수정 화면에서는 실제 원고와 목차 구조를 함께 관리합니다.  
페이지 제목, 부모 페이지, 공개 여부, 이미지, 본문 내용을 확인하고 필요한 부분을 수정합니다.

![](../assets/Pasted%20image%2020260429094221.png)

페이지 제목은 목차에 보이는 이름입니다.  
위키독스 목차는 같은 단계의 페이지를 이름순으로 정렬합니다.  
윈도우 탐색기에서 파일명이 이름순으로 보이는 것과 비슷하므로, 원하는 순서를 보장하려면 제목 앞에 `01`, `02`, `03-1`처럼 번호를 붙이는 방식이 좋습니다.

페이지의 상위 장과 하위 장 관계는 `부모 페이지` 항목에서 설정합니다.  
부모 페이지를 지정하면 해당 페이지가 그 페이지의 하위 장으로 들어가고, 부모 페이지를 지정하지 않으면 최상위 장으로 표시됩니다.

수정 화면의 왼쪽에는 현재 책의 목차가 표시됩니다.  
이 목차를 보면서 어떤 페이지가 있는지 확인하고, 수정할 페이지로 이동할 수 있습니다.

![](../assets/Pasted%20image%2020260429145719.png)

새 페이지를 만들 때는 왼쪽 목차 영역의 `신규 페이지 추가` 기능을 사용합니다.  
이 기능으로 페이지를 추가한 뒤 페이지 제목과 부모 페이지를 정하고 본문을 입력하면 책에 새로운 페이지가 생깁니다.

## 저장, 복제, 하위 페이지, 삭제

페이지 수정 화면 아래에는 페이지를 관리하는 버튼이 있습니다.  
저장은 현재 페이지의 변경 내용을 반영하고, 복제는 현재 페이지를 복사해 새 페이지를 만듭니다.

![](../assets/Pasted%20image%2020260429094331.png)

하위 페이지 추가 기능을 사용하면 현재 페이지 아래에 새 하위 장을 만들 수 있습니다.  
삭제는 현재 페이지를 제거하는 기능이므로 실행하기 전에 정말 삭제해도 되는지 확인해야 합니다.
