위키독스 MCP를 사용하려면 먼저 API 토큰을 준비해야 합니다.  
API 토큰은 LLM 도구가 내 위키독스 계정 권한으로 책과 페이지에 접근할 수 있게 해 주는 인증 정보입니다.

04-1장에서 위키독스 옵시디언 플러그인을 설정하며 API 토큰을 이미 확인했다면 같은 토큰을 사용하면 됩니다.  
아직 토큰을 확인하지 않았다면 이 장의 절차에 따라 준비합니다.

[TOC]

## API 토큰이 필요한 이유

MCP는 위키독스와 LLM 도구를 연결합니다.  
연결된 도구는 책 목록을 조회하고, 페이지 내용을 읽고, 필요한 경우 페이지를 새로 만들거나 수정할 수 있습니다.

이때 위키독스는 어떤 계정의 권한으로 작업하는지 확인해야 합니다.  
API 토큰은 이 확인에 사용되는 값입니다.

API 토큰은 비밀번호와 비슷하게 다루어야 합니다.  
토큰을 다른 사람에게 공유하면 내 계정으로 책이나 페이지를 수정할 수 있으므로 안전하게 보관해야 합니다.

## 토큰 발급 위치

위키독스에 로그인한 뒤 `계정설정` 화면으로 이동합니다.  
계정설정 메뉴에서 `API 토큰` 항목을 엽니다.

![](../assets/Pasted%20image%2020260429103739.png)

API 토큰 화면에서 `보기` 버튼을 선택하면 토큰 값을 확인할 수 있습니다.  
이 값을 복사해 MCP 서버 등록 과정에서 사용합니다.

![](../assets/Pasted%20image%2020260429103804.png)

화면 캡처를 남길 때는 토큰 값이 그대로 보이지 않도록 가리는 것이 좋습니다.  
책 원고나 예제 이미지에 실제 토큰이 들어가면 안 됩니다.

## 토큰을 사용할 때의 원칙

API 토큰은 설정 파일이나 환경 변수에 넣어 사용합니다.  
대화창에 그대로 붙여 넣는 방식은 피하는 것이 좋습니다.

특히 공개 저장소에 설정 파일을 올리는 경우에는 토큰이 포함되지 않았는지 반드시 확인합니다.  
토큰이 들어간 파일은 `.gitignore`에 추가하거나, 환경 변수로 분리해 관리합니다.

MCP 설정을 마친 뒤 연결이 제대로 되는지 확인했다면, 토큰 값 자체를 다시 열어 볼 일은 많지 않습니다.  
토큰을 복사해 둔 임시 메모가 있다면 설정이 끝난 뒤 삭제하는 것이 안전합니다.

## 접근 권한 이해하기

MCP로 접근할 수 있는 책은 내 계정이 관리할 수 있는 책입니다.  
내가 저자로 등록되어 있지 않은 책이나 권한이 없는 페이지는 조회하거나 수정할 수 없습니다.

따라서 연결 후 가장 먼저 할 일은 책 목록을 확인하는 것입니다.  
LLM 도구에 다음처럼 요청하면 접근 가능한 책을 확인할 수 있습니다.

```text
내 책 목록을 보여 주세요.
```

책 목록이 보인다면 토큰과 연결 권한이 정상적으로 동작하는 것입니다.  
목록이 비어 있거나 권한 오류가 난다면 로그인 계정, API 토큰, 책의 저자 권한을 차례대로 확인합니다.
