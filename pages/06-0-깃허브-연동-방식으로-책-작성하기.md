깃허브 연동 작성은 원고를 깃허브 저장소의 마크다운 파일로 관리하고, 변경된 내용을 위키독스 책에 반영하는 방식입니다. 위키독스 책과 깃허브 저장소를 연결해 두고, 원고 수정과 이력 관리는 깃허브를 기준으로 진행합니다.

이 방식은 옵시디언 같은 편집 도구와 Codex, Claude Code 같은 LLM 도구를 함께 활용하기 좋습니다. 원고를 파일로 다루면서 수정 이력을 남길 수 있고, 필요하면 이전 상태로 되돌리거나 다른 사람의 수정 제안을 검토할 수도 있습니다.

## 깃허브 연동 작성이란

깃허브 연동 작성은 위키독스 책과 깃허브 저장소를 연결해 원고를 관리하는 방법입니다. 책의 각 페이지는 `pages/` 폴더의 마크다운 파일로 작성하고, 책의 목차는 `TOC.md` 파일로 관리합니다.

위키독스에서 깃허브 연동 책을 만들면 책과 연결된 저장소가 자동으로 생성됩니다. 저자는 이 저장소를 로컬 컴퓨터로 내려받아 원고를 수정하고, 변경 내용을 깃허브에 push하여 위키독스에 반영합니다.

위키독스는 독자에게 책을 보여 주는 공간이 되고, 깃허브는 원고를 보관하고 변경 이력을 관리하는 공간이 됩니다.

## 깃허브 연동 작성을 사용하는 이유

깃허브를 사용하면 원고의 수정 이력이 자연스럽게 남습니다. 어떤 문장을 언제 바꾸었는지 확인할 수 있고, 필요하면 이전 상태로 돌아가거나 변경 내용을 비교할 수 있습니다.

공동 집필에도 유리합니다. 여러 사람이 함께 원고를 고치거나, pull request를 통해 수정 제안을 받고 검토한 뒤 반영하는 흐름을 만들 수 있습니다.

여러 도구와 함께 쓰기에도 좋습니다. 로컬 편집기에서 원고를 작성하고, LLM에게 초안을 만들게 하거나 문장을 다듬게 하고, 변경된 내용을 깃허브에 기록한 뒤 위키독스에 반영하는 흐름을 만들 수 있습니다.

책을 꾸준히 고쳐 나갈수록 이런 이력 관리의 장점은 더 커집니다. 원고를 하나의 프로젝트처럼 다루고 싶다면 깃허브 연동 작성이 특히 잘 맞습니다.

## 기본 작업 흐름

깃허브 연동 방식의 작업 흐름은 다음과 같습니다.

1. 위키독스에서 깃허브 연동 책을 만듭니다.
2. 위키독스가 자동으로 만든 깃허브 저장소를 확인합니다.
3. 저장소를 로컬 컴퓨터로 내려받아 옵시디언에서 엽니다.
4. 옵시디언 안에서 원고를 편집하고 LLM 도구를 함께 활용합니다.
5. 변경 내용을 깃허브에 push하고 위키독스 책 화면에서 결과를 확인합니다.
