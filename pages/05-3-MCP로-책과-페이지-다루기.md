MCP 서버 등록이 끝나면 LLM 도구에서 자연어로 위키독스 책과 페이지를 다룰 수 있습니다.

[TOC]

## 먼저 책 목록 확인하기

처음에는 내가 접근할 수 있는 책 목록을 확인합니다. LLM 도구에 다음처럼 요청합니다.

```text
내 위키독스 책 목록을 보여 주세요.
```

책 목록에는 보통 책 ID, 제목, 설명, 공개 여부 같은 정보가 표시됩니다. 이후 작업에서는 책 ID가 필요하므로, 수정할 책의 ID를 확인해 둡니다.

## 목차 구조 확인하기

책 ID를 알았다면 다음 단계는 목차 구조를 확인하는 것입니다. 큰 책에서는 전체 본문을 한 번에 읽기보다 목차만 먼저 보는 것이 효율적입니다.

```text
책 ID 123의 목차를 보여 주세요.
```

목차를 보면 어떤 페이지가 최상위 장이고, 어떤 페이지가 하위 장인지 확인할 수 있습니다. 수정할 페이지를 찾을 때도 먼저 목차를 확인하는 습관이 좋습니다.

## 필요한 페이지만 읽기

수정할 페이지를 찾았다면 해당 페이지 본문만 불러옵니다. 예를 들어 페이지 ID가 456이라면 다음처럼 요청합니다.

```text
페이지 ID 456의 내용을 보여 주세요.
```

큰 책 전체를 한 번에 읽게 하면 응답이 길어지고, LLM이 불필요한 내용을 함께 처리하게 됩니다. 따라서 책 구조는 목차로 확인하고, 본문은 필요한 페이지 단위로 읽는 방식이 좋습니다.

## 새 페이지 만들기

MCP를 사용하면 새 페이지도 만들 수 있습니다. 예를 들어 책 ID 123에 새 장을 추가하려면 다음처럼 요청할 수 있습니다.

```text
책 ID 123에 "1장. 시작하기"라는 페이지를 추가해 주세요.
내용은 위키독스 책 작성을 처음 시작하는 독자를 위한 짧은 안내문으로 작성해 주세요.
```

하위 페이지로 만들고 싶다면 부모 페이지도 함께 지정해야 합니다. 부모 페이지 ID를 알고 있다면 다음처럼 요청합니다.

```text
책 ID 123에 "1-1. 준비물" 페이지를 추가해 주세요.
부모 페이지는 페이지 ID 456으로 설정해 주세요.
```

## 페이지 수정하기

기존 페이지를 수정할 때는 바꿀 범위를 명확히 말하는 것이 중요합니다. 제목만 바꾸고 싶다면 제목만 바꾸라고 요청합니다.

```text
페이지 ID 456의 제목만 "1장. 위키독스 시작하기"로 수정해 주세요.
본문 내용은 바꾸지 마세요.
```

본문만 수정할 때도 마찬가지입니다.

```text
페이지 ID 456의 본문을 읽고 어색한 문장만 자연스럽게 다듬어 주세요.
제목과 공개 여부는 바꾸지 마세요.
수정 전후의 차이를 먼저 보여 준 뒤, 제가 승인하면 반영해 주세요.
```

공개 여부를 바꿀 때는 `공개` 또는 `비공개`를 분명히 말합니다.

```text
페이지 ID 456을 비공개로 바꿔 주세요.
```

부모 페이지를 제거해 최상위 페이지로 옮길 수도 있습니다.

```text
페이지 ID 456의 부모 페이지를 제거해서 최상위 페이지로 이동해 주세요.
```

## 책 집필에 활용하기

MCP는 초안을 만드는 데도 활용할 수 있습니다. 다만 한 번에 책 전체를 만들게 하기보다, 목차와 페이지 단위로 나누어 진행하는 것이 좋습니다.

예를 들어 다음처럼 요청할 수 있습니다.

```text
"파이썬 기초"라는 책의 목차 초안을 만들어 주세요.
각 장은 5개 이내로 구성하고, 초보자가 읽기 쉬운 순서로 정리해 주세요.
아직 위키독스에는 반영하지 말고 먼저 목차 초안만 보여 주세요.
```

목차를 확인한 뒤 페이지 생성을 요청합니다.

```text
방금 만든 목차 중 1장과 2장만 위키독스 책 ID 123에 페이지로 추가해 주세요.
각 페이지에는 짧은 소개 문단만 넣어 주세요.
```

## 사용할 수 있는 주요 도구

위키독스 MCP에는 여러 도구가 있지만, 책 작성에서 자주 쓰는 것은 다음과 같습니다.

| 도구 | 용도 |
| --- | --- |
| `list_books` | 내 책 목록 확인 |
| `create_book` | 새 책 생성 |
| `get_book_toc` | 책 목차와 위계 구조 확인 |
| `get_book` | 책 정보와 페이지 트리 확인 |
| `create_page` | 새 페이지 생성 |
| `get_page` | 특정 페이지 본문 조회 |
| `update_page` | 페이지 제목, 내용, 부모, 공개 여부 수정 |
| `upload_page_image` | 페이지에 사용할 이미지 업로드 |

MCP는 책 내용을 직접 수정할 수 있는 방식입니다. 중요한 페이지를 수정할 때는 먼저 현재 내용을 확인하고, 수정 범위를 작게 나누어 요청하는 것이 좋습니다.

API 토큰은 원고나 공개 저장소에 포함되지 않도록 주의합니다.
