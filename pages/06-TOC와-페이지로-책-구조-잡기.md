책은 글만 많다고 완성되지 않습니다. 위키독스에서는 `TOC.md`와 `pages/` 구조가 곧 책의 뼈대입니다.

## 기본 원칙

- `TOC.md`에 없는 페이지는 책 구조에 나타나지 않습니다.
- 페이지 순서를 유지하려면 제목 앞에 번호를 붙입니다.
- 파일명도 가능하면 제목 순서와 맞춰 두는 편이 좋습니다.

## 작업 순서

1. 먼저 목차의 큰 장을 정합니다.
2. 각 장에 대응하는 페이지 파일을 만듭니다.
3. `TOC.md`에 파일 경로를 연결합니다.
4. 필요한 이미지는 `assets/`에 추가합니다.
5. 페이지 안에서는 `../assets/...` 경로로 이미지를 참조합니다.

## 예시

```md
# 목차

- [01. 시작 전에 준비할 것](pages/01-시작-전에-준비할-것.md)
- [02. 전체 흐름 한눈에 보기](pages/02-전체-흐름-한눈에-보기.md)
```

## 자주 하는 실수

- `TOC.md` 경로와 실제 파일명이 다름
- 이미지 경로가 잘못됨
- 제목 번호와 파일 순서가 어긋남

다음 장에서는 이렇게 정리한 책을 실제로 위키독스에 반영합니다.
