원고는 깃허브 저장소에서 관리합니다.  
따라서 위키독스 계정과 함께 깃허브 계정도 필요합니다.

깃허브는 개발자만 사용하는 도구처럼 보일 수 있지만, 복잡한 기능을 모두 다루지는 않습니다.  
원고 파일을 보관하고, 변경 이력을 남기고, 위키독스와 연동하는 데 필요한 기본 기능만 사용합니다.

## 깃허브 가입하기

깃허브 계정이 없다면 먼저 깃허브에 가입합니다.

- [https://github.com](https://github.com)

![](../assets/Pasted%20image%2020260426121901.png)

가입할 때는 이메일 주소, 사용자 이름, 비밀번호가 필요합니다.  
사용자 이름은 깃허브에서 나를 나타내는 이름으로 사용되므로, 오래 사용할 수 있는 이름으로 정하는 것이 좋습니다.

가입 과정에서 이메일 확인을 요구할 수 있습니다.  
깃허브에서 보낸 확인 메일을 열어 인증을 마치면 계정을 사용할 수 있습니다.

## 로그인 확인하기

가입이 끝나면 깃허브에 로그인해 봅니다.  
로그인 후 자신의 프로필 화면이나 저장소 목록 화면으로 이동할 수 있다면 기본 준비는 끝난 것입니다.

![](../assets/Pasted%20image%2020260426121958.png)

이 단계에서 저장소를 직접 만들 필요는 없습니다.  
04장에서 위키독스와 연동할 책을 만들 때 필요한 저장소 구조를 함께 살펴봅니다.

## Git 설치 확인하기

깃허브 계정은 원격 저장소를 사용하기 위해 필요하고, Git은 내 컴퓨터에서 그 저장소를 내려받고 변경 내용을 올릴 때 필요합니다.  
뒤에서 터미널에서 `git clone`, `git commit`, `git push` 같은 명령을 사용하므로 Git이 설치되어 있어야 합니다.

터미널을 열고 다음 명령을 실행해 봅니다.

```bash
git --version
```

버전 번호가 표시되면 Git이 설치된 상태입니다.  
명령을 찾을 수 없다는 메시지가 나오면 Git을 설치해야 합니다.

- [https://git-scm.com/downloads](https://git-scm.com/downloads)
