앞 장에서는 위키독스가 자동으로 만들어 준 깃허브 저장소의 구조를 살펴보았습니다.  
이제 그 저장소를 내 컴퓨터로 내려받고, 옵시디언에서 열어 실제 원고 작업 공간으로 사용해 보겠습니다.

이 작업을 마치면 `pages/` 폴더의 원고를 옵시디언에서 바로 수정할 수 있고, 화면 캡처 이미지를 붙여 넣을 때도 `assets/` 폴더에 저장되도록 설정할 수 있습니다.

## 깃허브 저장소 주소 복사하기

먼저 GitHub에서 위키독스 책과 연결된 저장소 화면을 엽니다.  
저장소 화면에서 `Code` 버튼을 누르면 로컬로 내려받을 때 사용할 주소를 복사할 수 있습니다.

처음 따라 하는 경우에는 `HTTPS` 주소를 사용하는 편이 무난합니다.  
주소는 보통 다음과 같은 형태입니다.

```bash
https://github.com/사용자이름/저장소이름.git
```

![](../assets/Pasted%20image%2020260427100144.png)

## 로컬 컴퓨터로 저장소 내려받기

이제 터미널을 열고 책 원고를 보관할 위치로 이동합니다.  
예를 들어 여러 책 원고를 한곳에 모아 둘 생각이라면 `books` 같은 폴더를 만들어 그 안에서 작업해도 좋습니다.

원하는 위치로 이동한 뒤 `git clone` 명령으로 저장소를 내려받습니다.

```bash
git clone https://github.com/사용자이름/저장소이름.git
```

명령이 끝나면 저장소 이름과 같은 폴더가 만들어집니다.  
그 폴더 안에는 앞 장에서 살펴본 `README.md`, `TOC.md`, `pages/`, `assets/` 같은 파일과 폴더가 들어 있습니다.

비공개 저장소를 내려받는 경우에는 GitHub 로그인이 필요할 수 있습니다.  
터미널에 인증 안내가 표시되면 사용하는 GitHub 계정으로 로그인합니다.

이미 저장소를 내려받아 둔 적이 있다면 다시 `git clone`을 실행할 필요는 없습니다.  
그때는 기존에 내려받은 저장소 폴더를 그대로 사용하면 됩니다.

![](../assets/Pasted%20image%2020260427100504.png)

## 옵시디언에서 저장소 폴더 열기

옵시디언을 실행한 뒤 보관함을 여는 화면으로 이동합니다.  
새 보관함을 만드는 것이 아니라, 방금 내려받은 저장소 폴더를 기존 폴더로 열어야 합니다.

옵시디언 화면에서 `Open folder as vault` 또는 이와 비슷한 이름의 메뉴를 선택합니다.  
한국어 화면에서는 `폴더를 보관함으로 열기`, `기존 폴더 열기`처럼 표시될 수 있습니다.

파일 선택 창이 열리면 `git clone`으로 만들어진 저장소 폴더를 선택합니다.  
폴더를 열면 옵시디언 왼쪽 파일 목록에서 `README.md`, `TOC.md`, `pages`, `assets`를 확인할 수 있습니다.

![](../assets/Pasted%20image%2020260427100647.png)

![](../assets/Pasted%20image%2020260427100748.png)

[[TIP]]

옵시디언에서 저장소 폴더를 열면 `.obsidian/` 폴더가 생길 수 있습니다.  
이 폴더에는 개인별 옵시디언 설정이 저장되므로 보통 깃허브에 올리지 않습니다.

저장소의 `.gitignore` 파일에 다음 줄을 추가해 두면 `.obsidian/` 폴더가 깃에 포함되지 않습니다.

```text
.obsidian/
```

이미 `.obsidian/` 폴더가 깃에 등록된 상태라면 `.gitignore`에 추가하는 것만으로는 추적이 해제되지 않습니다.  
이때는 다음 명령으로 깃 추적 대상에서만 제거한 뒤 커밋합니다.

```bash
git rm --cached -r .obsidian
```

이 명령은 내 컴퓨터의 `.obsidian/` 폴더를 삭제하지 않고, 깃허브에 올릴 목록에서만 제외합니다.

[[/TIP]]

## 이미지가 assets 폴더에 저장되도록 설정하기

책을 작성하다 보면 화면 캡처를 원고에 자주 붙여 넣게 됩니다.  
이때 이미지 파일은 `assets/` 폴더에 저장하는 것이 좋습니다.

위키독스의 깃허브 연동 책은 저장소 안의 `assets/` 폴더를 이미지 파일을 보관하는 위치로 사용합니다.  
따라서 원고에서 사용할 화면 캡처나 삽화는 이 폴더에 모아 두는 편이 위키독스 반영 과정에서도 자연스럽습니다.

옵시디언 설정에서 새 첨부 파일의 저장 위치를 `assets`로 지정하면, 이미지를 붙여 넣거나 끌어다 놓을 때 파일이 자동으로 `assets/` 폴더에 저장됩니다.  
이렇게 해 두면 옵시디언에서 편하게 이미지를 붙여 넣으면서도, 위키독스가 기대하는 저장소 구조를 그대로 유지할 수 있습니다.

옵시디언의 설정을 열고 `파일 및 링크` 메뉴로 이동합니다.

그다음 `새 첨부 파일을 생성할 위치` 항목을 찾습니다.  
이 항목을 `아래에 지정된 폴더`로 바꾸고, `첨부 파일 폴더 경로`에 `assets`를 입력합니다.

정리하면 다음과 같이 설정합니다.

```text
새 첨부 파일을 생성할 위치: 아래에 지정된 폴더
첨부 파일 폴더 경로: assets
```

## 마크다운 링크 방식으로 저장하기

옵시디언은 기본적으로 `![[이미지.png]]`처럼 위키링크 형식으로 이미지를 넣을 수 있습니다.  
하지만 위키독스와 깃허브에서 함께 보기 좋은 원고를 만들려면 일반 마크다운 이미지 링크 형식을 사용하는 편이 좋습니다.

같은 `파일 및 링크` 설정 화면에서 `새로 생성하는 링크 형식` 항목을 찾습니다.  
이 항목은 `파일에 대한 상대 경로`로 설정합니다.

그 아래에 있는 `[[Wikilink]] 사용` 항목은 끕니다.  
이 옵션을 끄면 옵시디언이 `![[이미지.png]]` 형식 대신 일반 마크다운 이미지 링크를 만들어 줍니다.

정리하면 다음과 같이 설정합니다.

```text
새로 생성하는 링크 형식: 파일에 대한 상대 경로
[[Wikilink]] 사용: 끔
```

이렇게 해 두면 `pages/` 폴더의 원고에 이미지를 붙여 넣었을 때 다음과 같은 형태의 링크가 만들어집니다.

```md
![](../assets/Pasted%20image%2020260427012345.png)
```

이 형식은 현재 원고 파일을 기준으로 `assets/` 폴더의 이미지를 가리키므로, 파일 구조를 이해하기도 쉽고 깃허브에서도 확인하기 좋습니다.

![](../assets/Pasted%20image%2020260427101634.png)

## 이미지 붙여 넣기 확인하기

설정이 끝났다면 `pages/` 폴더 안의 원고 파일을 하나 열고 화면 캡처 이미지를 붙여 넣어 봅니다.  
이미지가 원고에 삽입되고, 왼쪽 파일 목록의 `assets/` 폴더 안에 이미지 파일이 생기면 설정이 제대로 된 것입니다.

이제부터 원고를 쓰다가 필요한 화면을 캡처해 바로 붙여 넣으면 됩니다.  
이미지 파일은 `assets/` 폴더에 모이고, 원고에는 해당 이미지를 불러오는 마크다운 링크가 남습니다.

![](../assets/Pasted%20image%2020260427101914.png)
