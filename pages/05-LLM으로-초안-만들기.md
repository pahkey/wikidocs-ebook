LLM은 책을 대신 완성해 주는 도구가 아니라, 초안을 빠르게 만들고 설명을 다듬는 보조 도구로 쓰는 편이 가장 실용적입니다.

## 언제 도움이 되는가

- 장의 개요를 먼저 잡고 싶을 때
- 이미 쓴 문단을 더 읽기 쉽게 바꾸고 싶을 때
- 예시, 체크리스트, 요약을 빠르게 만들고 싶을 때
- 같은 내용을 독자 수준에 맞게 다시 설명하고 싶을 때

## 간단한 사용 흐름

1. 이번 장의 주제와 독자 수준을 적습니다.
2. 포함해야 할 핵심 포인트를 정리합니다.
3. LLM에게 초안을 요청합니다.
4. 초안을 그대로 쓰지 말고 저자 관점으로 다시 다듬습니다.

## 예시 요청

```text
위키독스 책 만들기 입문자를 대상으로,
깃허브(GitHub) 연동 책의 장점을 3개의 소제목으로 설명해 줘.
광고 문구처럼 쓰지 말고 실무 가이드 톤으로 작성해 줘.
```

## 주의할 점

- 사실 여부는 직접 확인합니다.
- 내 경험과 판단이 빠지지 않도록 합니다.
- 책 전체 문체가 흔들리지 않게 다듬습니다.

다음 장에서는 이렇게 만든 원고를 실제 책 구조로 정리하는 방법을 살펴봅니다.
