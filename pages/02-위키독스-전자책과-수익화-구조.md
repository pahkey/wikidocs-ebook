# 02. 위키독스 전자책과 수익화 구조

위키독스에 책을 올리는 목적은 사람마다 다르지만, 많은 저자에게 중요한 질문은 결국 하나입니다.

`이 책이 단순한 기록을 넘어 실제 가치를 만들 수 있는가?`

이 장에서는 위키독스 전자책을 수익화 관점에서 바라볼 때 무엇을 먼저 이해해야 하는지 정리합니다.

## 수익화는 판매 이전에 신뢰에서 시작된다

전자책을 올렸다고 해서 바로 수익이 발생하지는 않습니다. 독자가 비용을 지불하는 이유는 파일 자체가 아니라, 저자가 정리한 경험과 구조와 시간을 구매하기 때문입니다.

따라서 먼저 점검할 것은 다음입니다.

- 독자가 명확하게 얻어갈 결과가 있는가
- 다른 무료 자료와 구분되는 관점이나 실전성이 있는가
- 업데이트를 이어갈 수 있는 주제인가

## 위키독스 전자책이 유리한 이유

- 이미 위키독스 안에 독자 유입 경로가 있습니다.
- 마크다운 기반이라 원고 수정 비용이 낮습니다.
- 한 번 발행하고 끝내는 책보다, 지속적으로 개정하는 책에 잘 맞습니다.

특히 기술서나 실무 가이드는 시간이 지나며 내용이 바뀌기 때문에, 개정과 재발행이 쉬운 구조가 중요합니다.

## 수익화 관점에서 생각해야 할 질문

- 무료 공개 범위와 유료 전자책 범위를 어떻게 나눌 것인가
- 독자가 돈을 내고 볼 만한 깊이와 구조를 제공하는가
- 업데이트 주기를 유지할 수 있는가
- 책 자체가 다른 활동으로 이어지는가

책의 수익은 판매액 하나로만 판단하지 않는 편이 좋습니다. 강의, 컨설팅, 커뮤니티 유입, 개인 브랜딩으로 이어질 수도 있기 때문입니다.

## 이 책에서 제안하는 실전 전략

이 책은 다음 전략을 권장합니다.

1. 먼저 작지만 완결된 목차를 만든다.
2. GitHub 연동 구조를 잡아 수정 비용을 낮춘다.
3. Obsidian에서 자료와 초안을 정리한다.
4. AI를 이용해 반복 작업 시간을 줄인다.
5. 발행 후에도 주기적으로 개정한다.

다음 장에서는 이 전략의 기반이 되는 GitHub 연동 작성 방식을 살펴봅니다.
