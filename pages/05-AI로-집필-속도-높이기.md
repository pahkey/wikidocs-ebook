# 05. AI로 집필 속도 높이기

AI는 책을 대신 써주는 도구라기보다, 집필 과정에서 반복 비용을 줄여주는 보조 도구로 보는 편이 정확합니다.

잘 쓰면 초안 작성, 문장 정리, 구조 재배치, 예시 생성에 큰 도움이 됩니다. 반대로 검토 없이 그대로 넣으면 책 전체의 밀도와 신뢰도가 떨어질 수 있습니다.

## AI에게 맡기기 좋은 작업

- 장의 개요 초안 만들기
- 이미 쓴 문단을 더 읽기 쉽게 재구성하기
- 독자 수준에 맞춰 설명 톤 조정하기
- 예시 목록이나 체크리스트 만들기
- 중복 표현과 어색한 문장 찾아내기

## 사람이 반드시 직접 판단해야 하는 부분

- 핵심 주장과 결론
- 실제 경험에서 나온 관찰
- 사실 여부와 최신성
- 책 전체의 관점과 흐름

AI가 생성한 문장을 그대로 누적하면 책이 평평해지기 쉽습니다. 따라서 저자의 경험, 실패 사례, 선택의 이유를 직접 넣는 것이 중요합니다.

## 추천 프롬프트 방식

좋은 요청은 보통 다음 요소를 포함합니다.

- 이 장의 독자가 누구인지
- 독자가 읽고 나서 무엇을 이해해야 하는지
- 유지해야 할 문체와 분량
- 반드시 포함할 사례나 제약

예를 들어 다음처럼 요청할 수 있습니다.

```text
위키독스 전자책 입문자를 대상으로,
GitHub 연동의 장점을 3개의 소제목으로 설명해 줘.
광고 문구처럼 쓰지 말고 실무 가이드 톤으로 작성해 줘.
각 소제목 아래에는 실제 작업에서 느끼는 장점 한 가지를 포함해 줘.
```

## AI 사용 원칙

- 초안은 빠르게 받되, 최종 문장은 직접 다듬습니다.
- 사실 검증이 필요한 내용은 따로 확인합니다.
- 여러 페이지에 같은 표현이 반복되지 않게 조정합니다.
- 책의 목소리는 저자가 유지합니다.
