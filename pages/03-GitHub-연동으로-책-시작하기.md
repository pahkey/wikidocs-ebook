# 03. GitHub 연동으로 책 시작하기

위키독스 GitHub 연동의 핵심은 간단합니다. 책의 원본은 GitHub 저장소에 두고, 위키독스는 그 결과를 동기화해 보여주는 출판 창구로 사용하는 것입니다.

![](../assets/github_wikidocs.jpg)

## 왜 GitHub 연동이 중요한가

GitHub 연동을 사용하면 책 집필이 단순한 웹 편집이 아니라 버전 관리가 가능한 문서 작업이 됩니다.

- 어떤 내용이 언제 바뀌었는지 추적할 수 있습니다.
- 잘못 수정했을 때 이전 상태로 되돌리기 쉽습니다.
- 여러 페이지를 한 번에 정리하기 좋습니다.
- AI 도구와 로컬 편집기를 연결하기 쉽습니다.

## 기본 저장소 구조

위키독스 GitHub 연동 책은 보통 다음 구조를 사용합니다.

```text
.
├── README.md
├── TOC.md
├── assets/
└── pages/
```

각 파일의 역할은 다음과 같습니다.

- `README.md`: 책 소개와 요약
- `TOC.md`: 목차와 페이지 계층 구조
- `pages/`: 실제 본문 페이지
- `assets/`: 이미지 파일

## 실무에서 지켜야 할 원칙

- 페이지를 만들거나 이름을 바꾸면 `TOC.md`도 같이 수정합니다.
- 제목 순서가 유지되도록 앞에 번호를 붙입니다.
- 페이지 안의 이미지는 `../assets/...` 경로로 참조합니다.
- 작은 수정도 자주 커밋해서 변경 이력을 남깁니다.

## 추천 작업 흐름

1. 로컬 편집기에서 원고를 수정합니다.
2. `TOC.md`와 페이지 링크가 맞는지 확인합니다.
3. 이미지를 `assets/`에 넣고 상대경로를 점검합니다.
4. 커밋 후 원격 저장소에 push 합니다.
5. 위키독스에서 자동 동기화 결과를 확인합니다.

동기화가 바로 보이지 않으면 위키독스의 GitHub 설정 화면에서 수동 동기화를 다시 시도하면 됩니다.
