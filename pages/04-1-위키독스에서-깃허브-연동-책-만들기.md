이제 위키독스에 새 책을 만들 차례입니다.  
여기서는 일반 책이 아니라 `깃허브 책 만들기` 기능을 사용합니다.

이 기능을 사용하면 위키독스 책과 깃허브 저장소가 함께 만들어집니다.  
먼저 위키독스에서 새 책을 만드는 화면으로 이동해 보겠습니다.

## 깃허브 책 만들기 메뉴로 이동하기

먼저 위키독스에 로그인합니다.  
로그인한 뒤 계정 설정 화면으로 이동하고, 나의 책 메뉴를 엽니다.

나의 책 화면에서 `깃허브 책 만들기` 버튼을 찾습니다.  
일반 책 만들기가 아니라 깃허브와 연동되는 책 만들기 기능을 선택해야 합니다.

![](../assets/Pasted%20image%2020260426203431.png)

## 책 정보 입력하기

깃허브 책 만들기 화면에서는 책을 만들기 위한 기본 정보를 입력합니다.  
여기에서 입력한 정보는 위키독스에 표시되는 책과 깃허브 저장소를 만드는 데 사용됩니다.

먼저 책 제목을 입력합니다.  
책 제목은 독자가 위키독스에서 보게 되는 이름이므로, 책의 주제가 잘 드러나도록 정하는 것이 좋습니다.

다음으로 연결 방식을 선택합니다.  
새 저장소를 만들 수도 있고, 이미 가지고 있는 저장소를 연결할 수도 있습니다.

처음부터 따라 하는 흐름이므로 새 저장소를 만드는 방식을 기준으로 진행합니다.  
새 저장소를 선택하면 위키독스가 입력한 이름으로 깃허브 저장소를 자동으로 만들어 줍니다.

![](../assets/Pasted%20image%2020260426203604.png)

## 리포지토리 이름과 공개 설정 정하기

리포지토리 이름은 깃허브에 만들어질 저장소의 이름입니다.  
보통 영문, 숫자, 하이픈, 언더스코어를 사용하여 정합니다.

예를 들어 `my-wikidocs-book`이나 `python-study-note`처럼 책의 내용을 짐작할 수 있는 이름을 사용할 수 있습니다.  
나중에 로컬 컴퓨터에서 저장소를 내려받아 작업하게 되므로, 너무 길거나 알아보기 어려운 이름은 피하는 것이 좋습니다.

공개 설정도 함께 선택합니다.  
공개 저장소로 만들면 깃허브에서도 원고 파일을 누구나 볼 수 있고, 비공개 저장소로 만들면 허용된 사용자만 볼 수 있습니다.

위키독스에서 책을 공개하더라도 깃허브 저장소 공개 여부는 별도로 생각해야 합니다.  
처음 연습하거나 원고를 준비하는 단계라면 비공개 저장소로 시작하는 것도 좋은 선택입니다.

## GitHub 연결 승인하기

책 정보를 입력한 뒤에는 위키독스와 GitHub를 연결해야 합니다.  
위키독스가 깃허브 저장소를 만들고, 이후 변경 내용을 동기화하려면 GitHub 권한 승인이 필요합니다.

`GitHub 연결하고 책 만들기` 버튼을 누르면 GitHub 로그인 화면이나 권한 승인 화면으로 이동합니다.  
사용할 GitHub 계정으로 로그인하고, 위키독스가 요청하는 권한을 확인한 뒤 승인합니다.

![](../assets/Pasted%20image%2020260426205111.png){width=50%}

권한을 승인하면 다시 위키독스로 돌아오고, 깃허브 연동 책 생성이 완료됩니다.  
이 과정에서 위키독스 책과 깃허브 저장소가 서로 연결됩니다.

## 자동으로 만들어지는 것

깃허브 연동 책을 만들면 몇 가지가 자동으로 준비됩니다.  
사용자가 저장소를 따로 만들거나 웹훅을 직접 설정할 필요는 없습니다.

먼저 GitHub 계정에 책 원고를 담을 저장소가 생성됩니다.  
이 저장소에는 책을 구성하기 위한 초기 파일과 폴더가 함께 만들어집니다.

또한 위키독스와 깃허브 저장소를 연결하는 설정도 만들어집니다.  
깃허브에 원고 변경 사항을 push하면 위키독스가 그 변경 내용을 감지해 책에 반영할 수 있는 기반이 마련됩니다.

![](../assets/Pasted%20image%2020260426205227.png)
