앞 장에서는 깃허브 저장소를 로컬 컴퓨터로 내려받고, 그 폴더를 옵시디언 보관함으로 여는 방법을 살펴보았습니다.  
이제 옵시디언 안에서 터미널을 열어 LLM 도구를 함께 사용하는 흐름을 만들어 보겠습니다.

이 방식으로 작업하면 옵시디언의 가운데 화면에서는 원고를 읽고 수정하고, 우측 화면에서는 Codex나 Claude Code 같은 도구를 실행하여 편집 도움을 받을 수 있습니다.

## 터미널 플러그인 설치하기

옵시디언에는 기본 터미널 기능이 포함되어 있지 않습니다.  
따라서 터미널을 옵시디언 안에서 사용하려면 커뮤니티 플러그인을 설치해야 합니다.

먼저 옵시디언 설정을 열고 `커뮤니티 플러그인` 메뉴로 이동합니다.  
처음 커뮤니티 플러그인을 사용하는 경우에는 제한 모드를 해제하고 커뮤니티 플러그인을 사용할 수 있도록 켭니다.

커뮤니티 플러그인은 옵시디언이 아닌 제3자가 만든 기능입니다.  
플러그인을 설치할 때는 이름과 설명, 제작자, 사용 후기를 확인하고 신뢰할 수 있는 플러그인만 설치하는 것이 좋습니다.

플러그인 탐색 화면에서 `Terminal`을 검색합니다.  
검색 결과에서 터미널, 셸, 콘솔을 옵시디언 안에 통합하는 플러그인을 선택한 뒤 설치하고 활성화합니다.

![](../assets/Pasted%20image%2020260427103009.png)

![](../assets/Pasted%20image%2020260427103047.png)

![](../assets/Pasted%20image%2020260427103155.png)


설치가 끝나면 옵시디언에 터미널을 여는 명령이나 아이콘이 추가됩니다.  
리본 아이콘을 눌러 열 수도 있고, 명령어 팔레트에서 `Terminal`을 검색해 열 수도 있습니다.

## 터미널 열기

여기서 여는 터미널은 macOS의 터미널 앱이나 윈도우 터미널이 아니라, 옵시디언 안에 표시되는 `Terminal` 플러그인의 터미널입니다.  
이 터미널을 사용하면 원고를 보면서 같은 화면 안에서 LLM 도구를 실행할 수 있습니다.

터미널 플러그인을 활성화하면 왼쪽 리본에 터미널 아이콘이 추가되거나, 명령어 팔레트에서 터미널 관련 명령을 실행할 수 있습니다.  
리본의 터미널 아이콘을 누르거나 명령어 팔레트에서 `Terminal`을 검색해 터미널을 엽니다.

![](../assets/Pasted%20image%2020260427103635.png)

터미널 프로필을 선택하는 화면이 나오면 옵시디언 안에 표시되는 통합 터미널 프로필을 선택합니다.  
외부 터미널을 여는 프로필을 선택하면 별도의 시스템 터미널 앱이 열릴 수 있으므로, 이 책의 흐름에서는 옵시디언 내부에 열리는 터미널을 사용합니다.

터미널이 열리면 옵시디언의 탭처럼 화면 안에 표시됩니다.  
이제 이 터미널이 책 저장소 폴더에서 열렸는지 확인합니다.

macOS나 리눅스 계열 셸을 사용하는 프로필에서는 다음 명령을 사용할 수 있습니다.

```bash
pwd
ls
```

윈도우 PowerShell 프로필을 사용하는 경우에는 다음 명령을 사용할 수 있습니다.

```powershell
cd
dir
```

터미널 위치가 책 저장소 폴더라면 `README.md`, `TOC.md`, `pages`, `assets` 같은 항목이 보입니다.  
이 위치에서 LLM 도구를 실행해야 원고 파일을 함께 살펴보며 작업할 수 있습니다.

만약 다른 폴더에서 터미널이 열렸다면 `cd` 명령으로 책 저장소 폴더로 이동합니다.

```bash
cd /path/to/your/wikidocs-book
```

## 터미널을 우측 패널로 이동하기

터미널을 원고 편집 화면과 같은 영역에 두면 화면이 좁아져 작업하기 불편할 수 있습니다.  
이럴 때는 터미널 탭을 오른쪽 패널로 옮겨 두면 좋습니다.

터미널 탭의 제목 부분을 잡고 옵시디언 오른쪽 사이드바 쪽으로 끌어 놓습니다.  
오른쪽 영역이 강조되면 마우스를 놓아 터미널을 그 위치에 배치합니다.

오른쪽 사이드바가 닫혀 있다면 먼저 오른쪽 사이드바를 펼칩니다.  
그다음 가운데에는 원고 파일을 열고, 오른쪽에는 터미널을 두면 편집 화면과 LLM 도구를 동시에 볼 수 있습니다.

![](../assets/Pasted%20image%2020260427104050.png)

[[TIP]]

터미널 패널이 너무 좁으면 오른쪽 패널의 경계를 드래그하여 폭을 조절합니다.  
LLM의 긴 답변을 읽어야 할 때는 잠시 터미널 폭을 넓히고, 원고를 직접 다듬을 때는 다시 줄이면 됩니다.

[[/TIP]]

## LLM 도구 실행하기

터미널 위치가 책 저장소 폴더인지 확인했다면 이제 LLM 도구를 실행합니다.  
사용하는 도구에 따라 실행 명령은 달라질 수 있습니다.

예를 들어 Codex를 설치했다면 다음과 같이 실행할 수 있습니다.

```bash
codex
```

Claude Code를 설치했다면 다음과 같이 실행할 수 있습니다.

```bash
claude
```

![](../assets/Pasted%20image%2020260427104821.png)

실행 후에는 현재 저장소의 원고 파일을 기준으로 도움을 요청합니다.  
처음부터 큰 범위를 한꺼번에 맡기기보다, 작은 단위로 요청하는 편이 결과를 확인하기 쉽습니다.

예를 들어 다음과 같이 요청할 수 있습니다.

```text
pages/04-3-저장소를-내려받아-옵시디언에서-열기.md 파일을 읽고 문장이 어색한 부분을 찾아 주세요.
```

```text
04장의 흐름에 맞춰 다음 절에 들어갈 설명 초안을 작성해 주세요. 파일은 아직 수정하지 말고 먼저 초안만 보여 주세요.
```

```text
TOC.md와 pages 폴더를 확인해서 목차 링크가 빠진 파일이 있는지 점검해 주세요.
```

## 편집 도우미로 사용하기

LLM 도구는 원고를 대신 완성하는 도구라기보다 편집 도우미로 사용하는 것이 좋습니다.  
초안을 만들고, 문장을 다듬고, 중복 설명을 찾고, 목차와 파일 연결을 확인하는 작업에 활용합니다.

터미널에서 LLM에게 요청하고, 결과는 옵시디언에서 직접 읽어 봅니다.  
마음에 드는 문장은 원고에 반영하고, 어색한 부분은 저자의 판단으로 다시 다듬습니다.

LLM이 파일을 직접 수정했다면 변경 내용을 바로 커밋하지 말고 먼저 확인합니다.

```bash
git diff
```

`git diff`로 어떤 문장이 바뀌었는지 확인한 뒤, 옵시디언에서 실제 원고를 다시 읽어 봅니다.  
이 과정을 거치면 LLM의 도움을 받으면서도 책의 문체와 방향을 저자가 계속 관리할 수 있습니다.
