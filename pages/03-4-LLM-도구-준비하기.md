이 책에서는 LLM 도구를 원고 작성의 보조 도구로 사용합니다.  
LLM 도구는 책을 대신 써 주는 도구라기보다, 초안을 만들고 문장을 다듬고 여러 파일을 함께 살펴보는 협업 도구에 가깝습니다.

특히 이 책에서는 옵시디언으로 원고를 열어 두고, 터미널에서 Codex나 Claude Code 같은 도구를 실행하는 흐름을 사용합니다.  
이렇게 하면 원고 파일이 있는 위치에서 바로 질문하고, 수정 방향을 요청하고, 변경 내용을 확인할 수 있습니다.

## 어떤 도구를 사용할까

대표적으로 사용할 수 있는 도구는 Codex와 Claude Code입니다.  
둘 다 터미널에서 실행할 수 있고, 현재 폴더의 파일을 읽어 원고나 코드 작업을 도와주는 방식으로 사용할 수 있습니다.

Codex는 OpenAI에서 제공하는 터미널 기반 도구입니다.  
Claude Code는 Anthropic에서 제공하는 터미널 기반 도구입니다.

이 책에서는 특정 도구 하나만 정답으로 정하지 않습니다.  
사용자가 이미 익숙한 도구가 있다면 그것을 사용해도 되고, 처음이라면 둘 중 하나를 선택해 시작해도 됩니다.

## 준비할 것

LLM 도구를 사용하려면 먼저 해당 서비스의 계정이 필요합니다.  
Codex를 사용하려면 OpenAI 또는 ChatGPT 계정이 필요하고, Claude Code를 사용하려면 Claude 또는 Anthropic 계정이 필요합니다.

또한 대부분의 터미널 기반 LLM 도구는 설치 과정에서 Node.js 같은 실행 환경을 요구할 수 있습니다.  
설치 방법은 도구와 운영체제에 따라 달라질 수 있으므로, 실제 설치 단계에서는 각 도구의 공식 문서를 확인하는 것이 좋습니다.

## 터미널에서 실행하기

이 책에서는 LLM 도구를 옵시디언 안의 터미널에서 사용하는 흐름을 다룹니다.  
옵시디언에서 책 원고 폴더를 열고, 같은 위치에서 터미널을 실행하면 LLM 도구가 원고 파일을 함께 살펴볼 수 있습니다.

예를 들어 원고가 들어 있는 폴더에서 LLM 도구를 실행한 뒤, 다음과 같은 요청을 할 수 있습니다.

```text
이 장의 문장을 더 자연스럽게 다듬어 주세요.
```

```text
03장과 03-1장의 설명이 겹치는 부분을 찾아 주세요.
```

```text
이 책의 문체에 맞춰 새 절의 초안을 작성해 주세요.
```

## 변경 내용을 확인하며 사용하기

LLM 도구가 원고 파일을 읽고 수정할 수 있다는 점은 편리합니다.  
하지만 LLM이 바꾼 문장을 그대로 받아들이기보다는, 저자가 직접 읽고 확인하는 과정이 필요합니다.

옵시디언은 원고를 읽고 다듬는 작업 공간으로 사용하고, 변경 내용 확인은 깃허브의 이력 관리 기능을 활용하면 좋습니다.  
LLM 도구가 파일을 수정한 뒤에는 터미널에서 `git diff`를 실행하여 어떤 문장이 바뀌었는지 확인할 수 있습니다.

처음에는 LLM에게 바로 파일을 고치게 하기보다 수정 방향이나 초안을 먼저 요청하는 방식이 좋습니다.  
파일을 수정한 뒤에는 옵시디언에서 실제 문장을 읽어 보고, 필요한 부분을 다시 다듬은 다음 마음에 드는 변경만 커밋합니다.
