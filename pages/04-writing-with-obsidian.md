# 04. Obsidian으로 원고 관리하기

GitHub가 원고의 기준 저장소라면, Obsidian은 집필과 아이디어 정리를 위한 작업 공간이 됩니다.

Obsidian의 장점은 단순히 마크다운을 편집할 수 있다는 점이 아닙니다. 책의 페이지를 독립된 노트처럼 다루면서도, 전체 구조를 한눈에 보며 초안과 메모를 빠르게 오갈 수 있다는 점이 더 중요합니다.

## Obsidian을 쓰는 이유

- 폴더 기반이라 GitHub 저장소와 잘 맞습니다.
- 마크다운 링크와 검색이 강력합니다.
- 초안, 메모, 인용문, 발췌를 빠르게 쌓을 수 있습니다.
- 긴 글을 쓰면서도 구조를 잃지 않기 쉽습니다.

## 권장 사용 방식

- `pages/` 아래의 실제 페이지 파일을 직접 편집합니다.
- 발행 대상이 아닌 개인 메모는 별도 폴더에 분리합니다.
- 장을 쓰기 전에 먼저 목차와 소제목만 잡아 둡니다.
- 한 페이지는 하나의 명확한 질문에 답하도록 작성합니다.

## 초안 작성 루틴 예시

1. `TOC.md`에서 이번에 쓸 장의 위치를 먼저 정합니다.
2. Obsidian에서 페이지 파일을 열고 소제목부터 만듭니다.
3. 메모와 참고 내용을 임시 문단으로 붙여 넣습니다.
4. 흐름이 잡히면 설명 문장을 다듬습니다.
5. 불필요한 중복을 줄이고 예시를 보강합니다.

## Obsidian과 GitHub를 함께 쓸 때 주의할 점

- 링크 자동완성이 파일명을 바꾸지 않도록 주의합니다.
- 발행용 문서와 개인 메모를 섞지 않습니다.
- 이미지 파일을 추가했다면 실제 경로가 `assets/`에 맞는지 확인합니다.
- 최종 반영 전에는 마크다운 미리보기와 Git diff를 함께 확인합니다.
