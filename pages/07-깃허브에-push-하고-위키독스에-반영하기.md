로컬에서 원고를 수정했다면 마지막 단계는 간단합니다. Git으로 커밋하고 깃허브(GitHub)에 push 하면 됩니다.

## 기본 반영 흐름

1. 변경 파일을 확인합니다.
2. `git add`로 반영할 파일을 고릅니다.
3. `git commit`으로 변경 내용을 기록합니다.
4. `git push`로 원격 저장소에 올립니다.
5. 위키독스에서 자동 동기화를 확인합니다.

## 동기화가 바로 안 보일 때

- push 가 정상적으로 되었는지 확인합니다.
- 저장소 웹훅이 살아 있는지 확인합니다.
- 위키독스 `책 수정 > 깃허브`에서 `지금 동기화`를 시도합니다.

## 왜 이 방식이 안정적인가

- 위키독스 웹 에디터보다 변경 이력이 명확합니다.
- 여러 페이지를 한 번에 묶어서 반영할 수 있습니다.
- 실수했을 때 이전 상태를 추적하기 쉽습니다.

다음 장에서는 발행 이후 운영과 수익화 관점을 간단히 정리합니다.
