# 위키독스 깃허브(GitHub) 연동 책

이 스킬은 깃허브(GitHub)를 통해 이미 위키독스와 연결된 책 저장소를 다룰 때 사용합니다.

## 전제

- 이 저장소가 책 내용의 기준 원본입니다.
- `git push` 후 위키독스가 변경사항을 받아 동기화합니다.
- 책은 아래 구조를 사용합니다.
  - `README.md`
  - `TOC.md`
  - `pages/`
  - `assets/`

사용자가 기존의 일반 위키독스 책을 깃허브(GitHub) 연동 책으로 바꾸고 싶어 해도, 이를 지원한다고 가정하지 않습니다. 공식 가이드의 2026-01-26 기준 설명에 따르면 깃허브(GitHub) 연동은 새 책 생성 시에만 가능하며, 이미 만든 책에 나중에 깃허브(GitHub)를 붙이는 방식은 지원되지 않습니다.

## 파일 역할

### `README.md`

- 책 요약 문서로 취급합니다.
- 첫 번째 `#` 제목은 책 제목이고, 그 아래 내용은 위키독스의 책 요약으로 사용됩니다.
- `README.md` 안의 이미지는 `./assets/example.png` 같은 루트 기준 상대경로를 사용합니다.

### `TOC.md`

- 책의 보이는 계층 구조를 정의하는 파일입니다.
- 페이지를 추가, 이름 변경, 이동, 삭제할 때마다 `TOC.md`를 함께 수정합니다.
- 형식은 아래처럼 유지합니다.

```md
# 목차

- [01. 시작하기](pages/01-getting-started.md)
  - [01-1. 설치하기](pages/01-1-install.md)
```

- 들여쓰기는 단계마다 2칸을 사용합니다.
- 원하는 순서를 유지하려면 제목의 번호 접두어를 유지합니다. 위키독스는 제목 기준 알파벳순으로 정렬합니다.

### `pages/*.md`

- 파일명은 가능하면 번호가 포함된 제목 구조를 반영합니다. 예: `01-getting-started.md`
- 이 저장소 규칙에서는 페이지 본문 맨 위에 H1 제목을 쓰지 않습니다. 페이지 제목은 `TOC.md`와 위키독스 구조에서 관리합니다.
- 새 페이지를 만들 때는 다음 순서를 따릅니다.
  1. `pages/` 아래에 마크다운 파일을 추가합니다.
  2. `TOC.md`에 같은 페이지를 등록합니다.
  3. 인접 챕터와 번호 체계를 맞춥니다.

### `assets/`

- 로컬 이미지는 이 폴더에 저장합니다.
- 페이지 파일에서는 `../assets/...` 경로로 이미지를 참조합니다.
- `README.md`에서는 `./assets/...` 경로를 사용합니다.
- 새 파일명은 가능하면 ASCII 기반 kebab-case를 우선합니다. 이미 동작 중인 한글 파일명은 굳이 바꾸지 않아도 됩니다.

## 작업 규칙

1. 페이지 구조를 바꾸기 전에 반드시 `TOC.md`를 먼저 확인합니다.
2. 깃허브(GitHub) 연동 책에서는 위키독스 웹 UI 편집을 기본 수정 경로로 취급하지 않습니다.
3. 구조 변경과 내용 변경이 한 작업에 속하면 같은 커밋에 묶습니다.
4. `TOC.md`에서 빠진 페이지는 동기화 후 책 구조에서도 사라질 수 있다고 봅니다.
5. 사용자가 내용 수정만 원하면 불필요한 파일명 변경이나 순서 재배치는 피합니다.

## 배포 흐름

1. 로컬에서 `README.md`, `TOC.md`, `pages/`, `assets/`를 수정합니다.
2. 상대 링크와 이미지 경로를 점검합니다.
3. 변경사항을 커밋합니다.
4. 연결된 깃허브(GitHub) 저장소로 push 합니다.
5. 위키독스가 push 이벤트를 받아 자동 동기화해야 합니다.

동기화가 보이지 않으면 먼저 아래를 확인합니다.

1. push 가 원격 저장소에 정상 반영되었는지 확인합니다.
2. 깃허브(GitHub) 저장소의 `Settings > Webhooks`를 확인합니다.
3. 위키독스의 `책 수정 > 깃허브`에서 연결 저장소와 웹훅 URL을 확인합니다.
4. `지금 동기화` 버튼으로 수동 재시도를 해봅니다.

## 자주 생기는 문제

- 페이지 순서가 이상함:
  - 제목 번호 체계나 `TOC.md` 순서가 어긋난 경우가 많습니다.
- 페이지가 보이지 않음:
  - `TOC.md`에 등록되지 않았거나 경로가 잘못된 경우를 먼저 의심합니다.
- 이미지가 깨짐:
  - 상대경로가 틀렸거나 파일명, 대소문자, 경로가 정확히 일치하지 않을 수 있습니다.
- 위키독스에서만 바로 수정하고 싶어 함:
  - 깃허브(GitHub) 연동 책의 제목, 내용, 구조는 저장소 기준으로 관리된다고 안내합니다.
